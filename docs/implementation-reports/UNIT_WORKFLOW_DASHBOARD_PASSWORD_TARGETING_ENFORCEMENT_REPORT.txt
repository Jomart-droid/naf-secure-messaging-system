Implemented corrections:
1. Dashboard role visibility
   - Dashboard now displays the logged-in user's Unit, Appointment and Workflow duty.
   - Added role-specific Unit Desk cards for Unit Workspace, Chief Clerk, AO, Signatory Officer, Commander and Other Officer.
   - Create/Draft button now appears only for users allowed to create signals; other unit roles are guided to Unit Workflow.

2. Unit user password management by unit main admin
   - Unit AO and Unit Workspace Account can update/reset passwords for users in their own unit only.
   - Super Admin/HQ Admin retain global override.
   - Commander/Chief Clerk/Other Officer do not get unit password-management authority.
   - Unit Users table now allows an optional new password; blank auto-generates a temporary password.
   - Reset forces password change at next login.
   - Audit event changed to UNIT_USER_PASSWORD_RESET.

3. Removed wrong unit/subordinate-only targeting restriction
   - Chief Clerk/unit signals may select any valid NAF unit/list/channel during drafting.
   - The selected routing is saved in the draft.
   - External delivery remains blocked until AO/signatory review and Commander approval/final release are complete.
   - Removed the old message: "You can only target your unit and subordinate units."

4. Audit tightening
   - Unit user creation/update/status/password events now use UNIT_USER_* audit names.
   - Unit workflow actions now log specific audit names such as UNIT_SIGNAL_SIGNED_BY_AO, UNIT_SIGNAL_FORWARDED_TO_SIGNATORY, UNIT_SIGNAL_APPROVED_BY_COMMANDER and UNIT_SIGNAL_RELEASED_BY_COMMANDER.

- AO/Signatory/Commander were still seeing dashboard wording that implied they could create/review signals the old way. Dashboard now routes them to their proper workflow desk.
- Password reset was only auto-generated. It now supports AO/Unit Workspace-provided password with policy validation.
- Old hierarchy restriction conflicted with distribution list/all-unit routing. It has been removed for Chief Clerk draft routing while preserving final workflow release gates.
