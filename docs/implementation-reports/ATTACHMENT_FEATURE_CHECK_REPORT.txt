   - Allowed UI extensions: PDF, Word, Excel, PowerPoint, text/CSV, images, ZIP.

2. Existing backend attachment save path is now activated for signals.
   - app/routes/messaging.py already had _save_broadcast_attachments(...).

3. Authorized download already exists and was verified in route code.
   - Route: /m/broadcasts/<broadcast_id>/attachments/<attachment_id>/download

4. Signal detail page already displays attachments.
   - Template: app/templates/messaging/broadcast_detail.html
   - Recipients can download attached files from the signal detail page.

5. Printable/exported signal now references attachment filenames.
   - Added _broadcast_attachment_print_block(...).
   - Print preview appends ATTACHMENT(S): filename list to the official TEXT section.
   - ReportLab PDF export path also appends the same attachment filename list.
   - Attached file contents are not embedded into the signal body; only filenames are listed.

6. Digital signature payload now includes attachment metadata.
   - Filename, SHA256, and size are included to strengthen integrity tracking.
